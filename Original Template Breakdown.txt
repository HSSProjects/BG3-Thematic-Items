

A lot of this is reworked or outdated but its a general idea on how the tool works, it 
just simplifies the process to a single form rather than dealing with many files and 
having to manually setup UIDs for everything.

If you know a bit more about the breakdowns on how different data pieces work or should
be used, feel free to leave a post or something so that I can implement it or add it to
the appropriate text files to help users out.





ModName.xml - 

<content contentuid="[UID_DESCRIPTION]" version="[VERSION]">[DESCRIPTION]</content> 
<content contentuid="[UID_NAME]" version="[VERSION]">[ITEM_NAME]</content>



merged.lsx - 

<node id="GameObjects">
	<attribute id="Description" type="TranslatedString" handle="[UID_DESCRIPTION]" version="[VERSION]" /> 
	<attribute id="DisplayName" type="TranslatedString" handle="[UID_NAME]" version="[VERSION]" /> 
	<attribute id="Icon" type="FixedString" value="[ICON_NAME]" />
	<attribute id="LevelName" type="FixedString" value="[LEVEL_NAME]" />
	<attribute id="MapKey" type="FixedString" value="[RootTemplateUUID]" />
	<attribute id="Name" type="LSString" value="[INTERNAL_NAME]" />
	<attribute id="ParentTemplateId" type="FixedString" value="[PARENT_TEMPLATE_UUID]" />
	<attribute id="Stats" type="FixedString" value="[INTERNAL_NAME]" />
	<attribute id="Type" type="FixedString" value="[ITEM_TYPE]" />
	<attribute id="_OriginalFileVersion_" type="int64" value="[FILE_VERSION]" />
	<children>
		<node id="GameMaster" />
	</children>
</node>



Armor.txt -

new entry "[INTERNAL_NAME]"
type "[ITEM_TYPE]"
using "[ITEM_TEMPLATE]"
data "RootTemplate" "[RootTemplateUUID]"
data "ValueOverride" "[VALUE]"
data "Weight" "[WEIGHT]"
data "Rarity" "[RARITY]"
data "Boosts" "[BOOSTS]"



Appendix A - Variables (ModName.xml)

    [DESCRIPTION]: Description of the magic item
    [ITEM_NAME]: Name of the item



Appendix B - Variables (merged.lsx)

    [UID_DESCRIPTION], [UID_NAME]: UID for the description and name in ModName.xml (use a random uuid generator example "h43576a52g162cg4eaeg9f13gf3bd5693f594")
    [PARENT_TEMPLATE_UUID]: Parent template UUID (seems to be a pointer to the item template we are copying from)
    [ICON_NAME]: Name of the item icon (need a selection ui for this) (F:\bg3\game_data\Public\Game\GUI\Assets\Tooltips\ItemIcons)
    [VERSION]: Version number (Semi-irrelevent, just use 1)
    [LEVEL_NAME]: Level name if any (idk what this is, just leave it empty)
    [FILE_VERSION]: File version number (idk what this is at all use this as default, if it works it works) (144115188075855912)
    


Appendix C - Variables (Armor.txt)

    [INTERNAL_NAME]: Internal name for the item
    [RootTemplateUUID]: Unique Identifier for the item  

    [ITEM_TEMPLATE]: Using which item template (_Ring)
    [ITEM_TYPE]: Type of item (Armor)

    [VALUE]: Item value override
	[WEIGHT]: Item weight
    [RARITY]: Item rarity {' "Uncommon', ' "Rare', ' "VeryRare', ' "Legendary', ' '}
    [BOOSTS]: Stat boosts given by the item
